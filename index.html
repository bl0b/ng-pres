ng-pres defines a number of new HTML elements and attributes to help write presentations in the most comfortable way.

It also implements the controls and mechanisms so you can go through the slides without the need for compiling, generating another file or using any other application but a browser.

The <presentation> tag is where it all happens.

The <slide> tag declares a new frame in which you put your contents.

As you have seen in the Hello World! example, the required HTML around the actual content is quite small—for HTML. Once both the Angular and ngPres scripts have been loaded, everything happens inside the <presentation> tag.

The contents of each individual slide is contained in a <slide> tag.

A little styling will be required to make it look good. We will cover that in a few slides.

Make an element visible only during the specified steps on a slide. This creates progressive slides. steps is a comma-separated list of ranges. The last step of a slide is the greatest number beginning a range. Ranges can be open on either side, but not both.

The element doesn't take any space and doesn't influence layout when it is not displayed.

Syntax

<some-element visible="RANGE_LIST">...</some-element>
Examples: 
<p visible="1">I appear only in step #1.<p>
<p visible="2-4">I appear in steps #2 to #4, included.<p>
<p visible="1,3-5">I appear in steps #1, #3, #4, and #5.<p>
<p visible="2-">I appear from step #2 on.<p>
<p visible="-3">I appear until step #3, included.<p>
<p visible="-2,4-">I disappear in step #3.<p>
