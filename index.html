ng-pres defines a number of new HTML elements and attributes to help write presentations in the most comfortable way.

It also implements the controls and mechanisms so you can go through the slides without the need for compiling, generating another file or using any other application but a browser.

The <presentation> tag is where it all happens.

The <slide> tag declares a new frame in which you put your contents.

As you have seen in the Hello World! example, the required HTML around the actual content is quite small—for HTML. Once both the Angular and ngPres scripts have been loaded, everything happens inside the <presentation> tag.

The contents of each individual slide is contained in a <slide> tag.

A little styling will be required to make it look good. We will cover that in a few slides.

You probably noticed this presentation stretches across the whole screen. Unlike most modern web things, the goal here is not exactly responsive design. You don't want the layout to adapt to various screen sizes, you want it to look exactly the same on any resolution, save for the size of the pixels.

This is achieved in ng-pres by automagically controlling the zoom factor of a fixed-size (yet vectorial) rendering, pretty much what happens when rendering a presentation from a PDF or *Office file.

So, the first thing to do is to set up the virtual frame size. By default, ng-pres assumes a 4:3 resolution of 1024x768, which is most common with video projectors. It seems that web browsers are much better at zooming out than at zooming in, so you may want to set up a size much bigger than the target screens.

CSS
table.presentation { width: 1024px; height: 768px; font-size: 16px; }

Five typical sizes are defined by default, ranging from huge to tiny. Redefine them at your leisure.

CSS
.huge { font-size: 1.5em; }
.big { font-size: 1.25em; }
.normal { font-size: 1em; }
.small { font-size: .75em; }
.tiny { font-size: .5em; }

If you are using the header, footer, or sidebars, you also want to style them at least to fix their sizes.

CSS
.footer { height: ***px; }
.header { height: ***px; }
.left-sidebar { width: ***px; }
.right-sidebar { width: ***px; }
.slide {}

Make an element visible only during the specified steps on a slide. This creates progressive slides. steps is a comma-separated list of ranges. The last step of a slide is the greatest number beginning a range. Ranges can be open on either side, but not both.

The element doesn't take any space and doesn't influence layout when it is not displayed.

Syntax

<some-element visible="RANGE_LIST">...</some-element>
Examples: 
<p visible="1">I appear only in step #1.<p>
<p visible="2-4">I appear in steps #2 to #4, included.<p>
<p visible="1,3-5">I appear in steps #1, #3, #4, and #5.<p>
<p visible="2-">I appear from step #2 on.<p>
<p visible="-3">I appear until step #3, included.<p>
<p visible="-2,4-">I disappear in step #3.<p>

Scale the contents to match the size of the adjustbox. Can maintain aspect ratio if desired. The adjustbox must be styled (either by CSS class or by using the style attribute) to the desired size, otherwise its size is 0. By default, stretch the contents to fill the adjustbox.

Syntax

<adjustbox attributes>
    contents...
</adjustbox>

Attributes

Name Value Effect
maintain-aspect none Makes it so the aspect ratio will be maintained. The contents will be centered in one direction and maxed in the other direction.
